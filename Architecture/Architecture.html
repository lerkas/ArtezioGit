Система управления запросами в IT-отдел


Система позволяет сотрудникам компании формировать запрос в IT-отдел, а системным администраторам и/или инженерам оперативно реагировать на них.


Предлагается использовать многоуровневую архитектуру.

Подключение юзеров к системе реализовано через web-интерфейс двух типов.

В локальной сети компании поднимается web-сервер (например Apache) + сервер приложений + база данных (например PostgreSQL).


Авторизация пользователей происходит посредством связки web-сервер + служба каталогов сети организации (например Apache + MS Active Directory).


Таким образом, если юзер уже залогинился в локальной сети организации, нет нужды второй раз вводить логин/пароль. Так же из AD берутся права доступа и определяется тип юзера.


Сотрудники компании формируют свой запрос через web-интерфейс первого типа (юзер без прав администрирования).


Запросы перенаправляются web-сервером на сервер приложений, где крутится некий(е) скрипт(ы) на JS и API, посредством которых запросы далее оседают в sql-базе данных.


Сотрудники IT-отдела через web-интерфейс второго типа (с правами администрирования), загружая запросы из базы данных, формируют из них очередь на выполнение.


После исполнения запроса, администраторы посредством web-интерфейса второго типа присваивают ему статус «выполнено», что влечёт за собой изменение записи в базе данных.


Сотруднику, инициировавшему запрос, приходит оповещение о выполнении.


Отказоустойчивость достигается путём бэкапа кластера через установленные промежутки времени.

Безопасность достигается путём интеграции с AD, как следствие — полностью соответствует групповым политикам доступа (при условии, что всё грамотно настроено, конечно).

Производительность — для работы системы со стороны пользователя необходим только браузер, потому есть возможность использования как тонких клиентов, переносимый устройств, так и традиционных ПК.
